# План реализации этапа 3 — Модуль ИИ и интерпретация пословиц

## Цель
Реализовать функционал интерпретации пословиц с помощью различных ИИ-моделей (PolzaAI), с возможностью сохранения истории, сравнения интерпретаций и управления через админ-панель.

## Задачи

### 1. Модели базы данных
- [ ] Создать таблицу `ai_model_config` для хранения параметров моделей ИИ
- [ ] Создать таблицу `ai_responses` для хранения ответов от моделей ИИ
- [ ] Обновить `models.py` с новыми моделями

### 2. Конфигурация и параметры
- [ ] Добавить переменные окружения для API-ключей моделей ИИ
- [ ] Реализовать загрузку конфигурации моделей из БД
- [ ] Создать CRUD-операции для управления моделями ИИ (админка)

### 3. Интеграция с провайдером PolzaAI
- [ ] Адаптировать `network.py` для работы с API PolzaAI
- [ ] Реализовать асинхронные запросы к ИИ
- [ ] Обработка ошибок и таймаутов

### 4. Обработка запросов
- [ ] Обработка нажатия кнопки "Анализ ИИ"
- [ ] Формирование промпта для интерпретации пословицы
- [ ] Выбор модели для анализа (по умолчанию или выбор пользователя)
- [ ] Отправка запроса и получение ответа
- [ ] Сохранение ответа в `ai_responses`

### 5. Работа с контекстом
- [ ] Сохранение истории интерпретаций для каждой пословицы
- [ ] Возможность продолжить диалог с ИИ по конкретной пословице
- [ ] Кэширование популярных интерпретаций (Redis или in-memory)

### 6. Безопасность и модерация
- [ ] Фильтрация небезопасного контента в ответах ИИ
- [ ] Лимиты запросов к ИИ-моделям (по пользователю)
- [ ] Модерация пользовательских комментариев к интерпретациям

### 7. Администрирование
- [ ] Команды `/models`, `/responses` для просмотра статистики ИИ
- [ ] Возможность добавить/удалить модель ИИ через админку
- [ ] Очистка истории ответов ИИ

### 8. Тестирование и документация
- [ ] Написание unit-тестов для модуля ИИ
- [ ] Обновление `DATABASE.md` с новыми таблицами
- [ ] Документация использования API PolzaAI

## Результат
Пользователи могут:
- Получать интерпретацию пословиц от ИИ
- Сравнивать интерпретации от разных моделей
- Просматривать историю анализа
- Администраторы могут управлять моделями и просматривать логи

Бот поддерживает расширяемую архитектуру для подключения новых ИИ-провайдеров.
